# Git コミットメッセージ作成ガイド

## コミットメッセージのルール

- 対応 issue がある場合は、コミットメッセージに `#<issue番号>` を記載する
- Issue 番号は現在のブランチ名から取得できる
  - 例: `feature/issue7/add-docs` の場合は `7` が Issue 番号

## コミット / プッシュの実行ポリシー

`git commit` と `git push` は **ユーザーの明示的な許可が必要** なオペレーションである。

- ユーザーから明示的に「コミットしてください」「プッシュしてください」と指示があるまでは、絶対に実行しない
- ユーザーが過去の会話でコミットを許可した場合でも、その許可は当該作業に限定される
- 別の作業に移ったら、改めて許可を取る

## 実行例

許可を得た後の実行：

```bash
git commit -m "$(cat <<'EOF'
コミットメッセージの本文 #451
EOF
)"
```
